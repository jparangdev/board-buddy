## 1. 회원 가입 및 계정 관리

### 시나리오 1-1: 정상 회원 가입

**목표**: 신규 사용자가 이메일/닉네임/비밀번호로 계정을 생성한다.

**사전 조건**: 미인증 상태

**흐름**:
1. `/register` 페이지 접속
2. 이메일, 닉네임, 비밀번호, 비밀번호 확인 입력
3. "계정 만들기" 클릭
4. `/login` 페이지로 리다이렉트

**결과**: 계정 생성 완료. `닉네임#XXXX` 형태의 고유 태그 자동 부여 (discriminator는 서버가 할당).

---

### 시나리오 1-2: 중복 이메일 방어

**목표**: 이미 사용 중인 이메일로 재등록 시도 시 에러를 표시한다.

**흐름**:
1. 이미 가입된 이메일로 `/register` 제출
2. `"Email already in use: {email}"` 에러 메시지 노출
3. 페이지 유지 (리다이렉트 없음)

---

### 시나리오 1-3: 닉네임 변경

**목표**: 로그인한 사용자가 닉네임을 변경한다.

**흐름**:
1. 사용자 메뉴 → 프로필 / 계정 설정 접속
2. 새 닉네임 입력 후 저장
3. 닉네임은 변경되지만 discriminator(#XXXX)는 고정 유지

---

## 2. 로그인 / 로그아웃

### 시나리오 2-1: 이메일/비밀번호 로그인

**흐름**:
1. `/login` 페이지 접속
2. 이메일 + 비밀번호 입력 후 "로그인" 클릭
3. `/groups` 페이지로 리다이렉트

---

### 시나리오 2-2: 인증 실패 (잘못된 비밀번호)

**흐름**:
1. 올바르지 않은 비밀번호 입력
2. 에러 메시지 노출, 페이지 유지

---

### 시나리오 2-3: 로그아웃

**흐름**:
1. 상단 네비게이션 → "로그아웃" 클릭
2. `/login` 페이지로 리다이렉트
3. 보호된 페이지 직접 접근 시 `/login`으로 리다이렉트

---

## 3. 게임 목록 확인 및 게임 등록

### 시나리오 3-1: 공식 게임 목록 확인

**목표**: 서버에 사전 등록된 공식 게임을 확인한다.

**흐름**:
1. 상단 메뉴 → `/games` 접속
2. 공식 게임 목록 노출 (예: Catan, Splendor, Love Letter, Hanabi 등)
3. 각 게임 카드에 최소/최대 플레이어 수 표시

---

### 시나리오 3-2: 새 공식 게임 등록

**목표**: 서비스에 없는 보드게임을 글로벌 목록에 추가한다.

**흐름**:
1. `/games` 페이지 → "+ 게임 추가" 클릭
2. 게임 이름, 최소/최대 플레이어 수, 점수 전략 선택:
   - `HIGH_WIN`: 높은 점수 승리
   - `LOW_WIN`: 낮은 점수 승리
   - `RANK_ONLY`: 순위만 기록
   - `WIN_LOSE`: 승/패 기록
   - `COOPERATIVE`: 협동 게임 (팀 전체 승/패)
3. 저장 → 게임 목록에 추가됨
4. 동일 이름 중복 등록 시 에러

---

## 4. 모임(Group) 생성

### 시나리오 4-1: 모임 생성 (멤버 즉시 초대)

**목표**: 새 보드게임 모임을 만들고 멤버를 추가한다.

**흐름**:
1. `/groups` → "+ 모임 만들기" 클릭
2. 모임 이름 입력
3. 멤버 검색: `닉네임#태그` 또는 **이메일 주소**로 검색 → 결과 클릭하여 선택
4. "모임 생성" 클릭 → `/groups` 리다이렉트
5. 생성된 모임 클릭 → 멤버 목록에서 본인(오너) + 선택한 멤버 확인

> **정책**: 모임은 불변(Immutable). 멤버 구성 변경이 필요하면 새 모임 생성.

---

### 시나리오 4-2: 모임 목록 순서 변경 (드래그 앤 드롭)

**흐름**:
1. `/groups` 페이지에서 모임 카드 드래그
2. 원하는 위치에 드롭
3. 페이지 새로고침 후에도 순서 유지 확인

---

## 5. 모임 초대 — 초대자 흐름

### 시나리오 5-1: 기존 멤버에게 초대장 발송

**목표**: 이미 생성된 모임에 새 멤버를 초대한다.

**사전 조건**: 모임 오너 또는 멤버 권한

**흐름**:
1. 모임 상세 페이지 (`/groups/:id`) 접속
2. "멤버 초대" 또는 유사한 버튼 클릭
3. `닉네임#태그` 또는 **이메일 주소**로 유저 검색
4. 검색 결과에서 대상 유저 선택 → 초대 전송
5. 초대 발송 확인 메시지 표시

---

## 6. 모임 초대 — 피초대자 흐름

### 시나리오 6-1: 초대 수락

**목표**: 받은 초대를 수락하여 모임에 합류한다.

**흐름**:
1. 상단 네비게이션 → "초대함" (`/invitations`) 접속
2. 받은 초대 목록 확인 (모임 이름, 초대한 사람, 날짜 표시)
3. 해당 초대의 "수락" 버튼 클릭
4. 초대 목록에서 해당 항목 사라짐
5. `/groups` 에서 합류한 모임 확인 가능

---

### 시나리오 6-2: 초대 거절

**흐름**:
1. `/invitations` 접속
2. 받은 초대에서 "거절" 버튼 클릭
3. 목록에서 해당 항목 제거

---

### 시나리오 6-3: 초대함 빈 상태

**흐름**:
1. `/invitations` 접속
2. 대기 중인 초대가 없을 때 빈 상태 메시지 노출
3. "모임 목록으로 돌아가기" 버튼 제공

---

## 7. 게임 세션 기록

게임 기록은 **4단계 위저드**로 진행된다.

### 시나리오 7-1: 점수 기반 게임 (HIGH_WIN / LOW_WIN)

**예시**: Catan (HIGH_WIN, 3인 플레이)

**흐름**:
1. 모임 상세 페이지 → "게임 기록" 클릭
2. **Step 1 — 게임 선택**: 공식 게임 목록에서 `Catan` 선택
3. **Step 2 — 플레이어 선택**: 참여 멤버 체크박스로 선택 (최소 1명)
4. **Step 3 — 결과 입력**:
   - 점수 전략 확인 (`HIGH_WIN`)
   - 플레이 일시 설정 (기본값: 현재 시각)
   - 각 멤버 점수 입력
5. **Step 4 — 확인**: 게임명, 플레이 일시, 플레이어+점수 요약 확인
6. "저장" → 모임 상세 페이지로 이동
7. 세션 목록에 기록 반영

---

### 시나리오 7-2: 순위 기반 게임 (RANK_ONLY)

**흐름**:
1. 게임 선택 → 점수 전략 `RANK_ONLY` 설정
2. 플레이어 선택
3. 결과 입력 단계에서 **드래그 앤 드롭으로 순위 지정**
4. 우승자 수(winnerCount) 설정
5. 확인 → 저장

---

### 시나리오 7-3: 승/패 기반 게임 (WIN_LOSE)

**흐름**:
1. 점수 전략 `WIN_LOSE` 선택
2. 승점/패점 입력 (기본: 승 +3점, 패 +0점)
3. 각 플레이어별 토글 버튼으로 WIN / LOSE 설정
4. 확인 → 저장

---

### 시나리오 7-4: 협동 게임 (COOPERATIVE)

**흐름**:
1. 점수 전략 `COOPERATIVE` 선택
2. 팀 전체의 승/패 결과 토글로 설정
3. 팀 승리 시 전원에게 동일한 승점 부여, 패배 시 패점 부여
4. 확인 → 저장

---

### 시나리오 7-5: 커스텀 게임 생성 후 세션 기록

**목표**: 목록에 없는 게임을 모임 전용 커스텀 게임으로 등록하고 바로 기록한다.

**흐름**:
1. 게임 선택 단계 → "+ 커스텀 게임 추가" 클릭
2. 모달에서 게임 이름, 최소/최대 플레이어 수, 점수 전략 입력 → "추가"
3. 커스텀 게임 목록에 나타남 → 선택
4. 이후 시나리오 7-1~7-4와 동일하게 진행

> **정책**: 커스텀 게임은 해당 모임에서만 표시됨.

---

### 시나리오 7-6: 일부 멤버만 참여한 세션

**흐름**:
1. 모임 내 전체 멤버 중 일부만 선택
2. 선택된 멤버에 대해서만 결과 입력
3. 저장 → 선택된 멤버에게만 결과 반영

---

## 8. 모임 대시보드(통계) 조회

### 시나리오 8-1: 통계 페이지 이동

**흐름**:
1. 모임 상세 페이지 → "통계 보기" 버튼 클릭
2. `/groups/:id/dashboard` 로 이동
3. 상단에 모임 이름 표시
4. "← 모임으로 돌아가기" 링크로 복귀

---

### 시나리오 8-2: 세션이 있는 모임 통계

**표시 항목**:

| 섹션 | 내용 |
|---|---|
| **요약 카드** | 총 세션 수, 총 플레이 횟수 |
| **가장 많이 플레이 (🎮)** | 참여 세션 수 기준 랭킹 + 진행 막대 |
| **가장 많이 이긴 (🏆)** | 승리 횟수 기준 랭킹 + 진행 막대 |
| **승률 (📈)** | 3게임 이상 참여 플레이어만 표시, 백분율 |
| **누적 점수 랭킹** | 플레이어별 누적 점수 합계 |
| **인기 게임 (🎲)** | 모임에서 가장 많이 플레이된 게임 |

**승률 최소 게임 수 정책**: 3경기 미만 플레이어는 "Need 3+ games" 메시지 표시.

---

### 시나리오 8-3: 세션이 없는 모임 통계 (빈 상태)

**흐름**:
1. 세션이 없는 모임의 "통계 보기" 클릭
2. 랭킹 섹션 없이 "아직 게임 기록이 없습니다" 메시지 노출
3. "게임 기록하기" CTA 버튼 → 세션 기록 페이지로 이동

---

## 9. 게임 세션 상세 조회

### 시나리오 9-1: 세션 목록 및 상세 확인

**흐름**:
1. 모임 상세 페이지에서 기록된 세션 목록 확인
2. 세션 클릭 → 상세 페이지 이동
3. 게임명, 플레이 일시, 참여자별 점수/순위/승패 확인

---

## 10. 계정 삭제

### 시나리오 10-1: 계정 삭제

**흐름**:
1. 사용자 메뉴 → "계정 삭제" 클릭
2. 삭제 확인 다이얼로그 표시
3. 확인 → 계정 삭제 및 `/login` 리다이렉트

## 부록: 주요 스코어 전략 요약

| 전략 | 설명 | 랭킹 계산 |
|---|---|---|
| `HIGH_WIN` | 높은 점수가 승리 | 점수 내림차순 → 1위 |
| `LOW_WIN` | 낮은 점수가 승리 | 점수 오름차순 → 1위 |
| `RANK_ONLY` | 점수 없이 순위만 | 드래그 순서 기반 |
| `WIN_LOSE` | 승/패 플래그 | 승리 여부 기반 |
| `COOPERATIVE` | 팀 전체 승/패 | 팀 결과 동일 적용 |

**동점자 처리**: 동일 점수 → 동일 순위 부여, 다음 순위는 건너뜀 (예: 공동 1위 2명이면 다음은 3위).

| `/games` | 공식 게임 목록 및 등록 |
| `/invitations` | 받은 모임 초대 목록 |
